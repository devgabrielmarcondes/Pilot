§1 — Foundation

Visual Direction

An AI operations tool, not a marketing page. The interface is a monochrome shell with a single purple accent for interactions — color exists only where it communicates state, data, or agent activity.

Design Principles
PrincipleMeans
Monochrome + accentPure black (#000) + pure white (#fff) for all interface elements. Single purple accent (#7C3AED) for hovers, focus rings, and active states. No other chroma in the shell.
Color = signalSemantic color appears only for agent states (running, success, error, waiting-approval), data visualization, and status badges. Never decorative.
Variable weight hierarchySpace Grotesk for display (300–700) at bold, attention-grabbing sizes. Inter for body (300–600) at precise weight stops: 350 (light body), 400 (body), 450 (medium), 500 (subheadings), 600 (headings). Hierarchy through weight + size together.
Density over decorationCompact information density. 8px card radius, tight spacing, mono for metadata. Every pixel earns its place.
Pill geometryAll interactive elements use pill (9999px) or circle (50%) radius. Cards and containers use 8px. No mixing.
Solid focus ringAll focus states use solid 2px outlines in the accent purple — clear, accessible, and consistent across every interactive element.
What This Is NOT
No gradient blobs No glassmorphism No AI orbs / sparkles No oversized hero No emoji icons No warm beige / peach wash No landing-page energy

Rules
  • The chrome layer is monochrome with a single purple accent. Purple (#7C3AED) appears only for interactive states: button hovers, focus rings, active tabs. No other chroma in the shell.
  • Signal colors appear only inside components — badges, timeline dots, progress bars, chart fills. Never as page backgrounds or card tints.
  • Tinted backgrounds for badges use 10% opacity of the signal color on white. Never solid-color badge backgrounds.
  • Data visualization (charts on Evaluations page) may use the full signal palette plus 2 additional neutrals for series differentiation.

Layout Decisions
DecisionRationale
Left panel: brief form + quick stats (280px fixed)The brief is the input context — always visible so the user can reference it while scanning the timeline. Quick stats (step count, tool count, elapsed time) give at-a-glance progress.
Right panel: timeline → traces → approval → report (scrollable)Chronological flow matches how the user experiences the agent. Timeline at top (most recent activity), traces collapsible below, approval gates inline, final report at bottom.
Report uses pill tabs, not underlined tabsPill tabs match the Figma-inspired geometry system. Underlined tabs feel like a website. Pill tabs inside a card feel like a tool.
Approval gates are inline cards, not modalsModals break the timeline flow and obscure context. An inline card with a purple left border keeps the user oriented in the sequence.
Tool-call traces are collapsed by defaultMost users don’t need to see JSON. The collapsed header shows tool name + status + duration — enough for scanning. Expand for debugging.

§10 — Delivery

UX & Copy Improvements

Making the agentic flow legible to a portfolio viewer who has 30 seconds to understand what this does.

Copy Principles
PrincipleExample
Name the action, not the technology“Searched 847 creators” not “Executed search_creators tool”
Show the number“Filtered to 24 matches” not “Results found”
State the duration“2.1s” in mono next to every completed step. Running steps show “running...” with pulse animation.
Explain the gate“Review the 24-creator shortlist before outreach proceeds” not “Human approval required”
Name the guardrail“Brand safety: competitor mention blocked” not “Guardrail triggered”
Use active voice for the agent“The agent generated a 6-week calendar” not “Calendar was generated”
Agentic Flow — Making It Legible
ProblemSolution
Viewer can’t tell what the agent is doing right nowRunning step has a pulsing blue dot + “running...” text. All previous steps show green checkmarks. Future steps show grey/purple dots. The visual state is instantaneously scannable.
Tool calls look like debug outputCollapse by default. Header shows: tool name (mono), status badge, duration. Expand to see JSON. Most portfolio viewers won’t expand — the header is enough.
Human-in-the-loop feels like a bug (why did it stop?)Purple left-border card with explicit language: “The agent paused because it needs your approval on X. Approve to continue.” The purple color is unique to approval — it never appears elsewhere.
Guardrail blocks look like errorsUse red dot + “blocked” language, but frame it positively: “The guardrail caught a competitor mention and removed it.” This is a feature, not a failure.
Final report feels disconnected from the processThe report appears at the bottom of the timeline, below all steps. The tab structure (Strategy / Calendar / Creators / Safety) maps to the agent’s work. The Safety tab shows guardrail results — connecting back to the timeline.

§11 — Delivery

Implementation Checklist

Prioritized for a developer working quickly. P0 items transform the visual quality. P1 items add credibility. P2 items are polish.

  • P0 Set --radius: 9999px in shadcn globals for buttons; keep rounded-lg (8px) for cards. This single change shifts the geometry from “generic SaaS” to “designed tool.”
  • P0 Replace all colored backgrounds with monochrome chrome. Remove any gradient backgrounds, tinted card surfaces, or colored sidebars. White background, black text, black borders.
  • P0 Switch display font to Space Grotesk (next/font/google) at weights 300–700 for headings and page titles. Use Inter for body at 300–600. Add JetBrains Mono for all metadata, timestamps, and numeric values.
  • P0 Implement the two-column workspace layout: 280px fixed left panel (brief + stats), fluid right panel (timeline + traces + approval + report). Collapse to single column below 1024px.
  • P0 Build the agent timeline as a custom component with the 5 dot states (done/running/waiting/error/pending). Use the exact colors and sizes from §6.
  • P0 Replace underlined tabs with pill-style tabs (black active, transparent inactive, inside a 4% black container with pill radius). Apply to report tabs and any nav tabs.
  • P0 Style all badges as 10%-opacity tinted pills with mono text. Remove any solid-color badges or rectangular badges.
  • P1 Add solid purple focus outlines to all interactive elements: focus-visible:outline-solid focus-visible:outline-2 focus-visible:outline-violet-600. Remove any default blue ring. Add creative hover transitions (0.3s cubic-bezier, translateY, glow shadow).
  • P1 Build tool-call trace as a collapsible component (shadcn Collapsible). Header: status badge + mono tool name + mono duration. Body: JSON in mono on 3% black background.
  • P1 Style approval gates as inline cards with 3px left border in purple (#9333ea). Two buttons: “Approve & Continue” (black pill) + “Request Changes” (white outline pill).
  • P1 Evaluations page: 5-up metric cards (mono number + label), detail table below, 2-column scenario cards. No charts needed — the table is more credible.
  • P1 Add pulse animation to the running-state timeline dot: opacity 0.4→1→0.4 over 1.5s infinite.
  • P1 Apply copy improvements from §10: name the action, show the number, state the duration, explain the gate.
  • P2 Add subtle hover states to timeline items (slight background tint on hover) to signal expandability.
  • P2 Implement mobile bottom tab bar (Workspace / Evaluations) for navigation below 768px.
  • P2 Add keyboard shortcuts: Cmd+Enter to approve, Esc to collapse traces.
  • P2 Persist campaign state to localStorage so a refresh during the portfolio video doesn’t lose the demo state.
